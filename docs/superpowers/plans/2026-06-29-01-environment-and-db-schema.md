# Environment Setup + DB Schema — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Spin up local development environment and create the Prisma DB schema with all v1 tables across 4 PostgreSQL schemas.

**Architecture:** Supabase CLI runs a local PostgreSQL instance via Docker. NestJS backend lives in `api/`. Prisma uses the `multiSchema` preview feature to manage tables across `user`, `topic`, `media`, and `chat` schemas. A shared `PrismaService` wraps the client with query logging enabled.

**Tech Stack:** Docker Desktop, Supabase CLI, Node.js 20+, NestJS CLI, Prisma 5+, TypeScript

## Global Constraints

- Node.js >= 20
- TypeScript strict mode enabled
- All tables in named schemas — never `public`
- Prisma multiSchema preview feature must be enabled
- PrismaClient must log `['query', 'info', 'warn', 'error']` in development
- `DATABASE_URL` and `DIRECT_URL` both required (Supabase requires both for Prisma)
- Repo structure: `api/` for backend, `mobile/` for app (created in a later plan), `supabase/` for Supabase CLI config

          ? ['warn', 'error']

  async onModuleDestroy() {

  }
}

### Task 5: Write Prisma schema — all v1 tables

**Files:**
- Modify: `api/prisma/schema.prisma`

**Interfaces:**
- Produces: All Prisma models used by feature modules in later plans:
  - `prisma.account` — `user.accounts` table
  - `prisma.topic` — `topic.topics` table
  - `prisma.topicFollow` — `topic.follows` table
  - `prisma.photo` — `media.photos` table
  - `prisma.photoLike` — `media.photo_likes` table

- [ ] **Step 1: Write the full Prisma schema**

Replace the full contents of `api/prisma/schema.prisma`:
```prisma
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["multiSchema"]
}

datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
  schemas   = ["user", "topic", "media", "chat"]
}

// ─── user schema ──────────────────────────────────────────────

model Account {
  id           String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  email        String   @unique
  passwordHash String   @map("password_hash")
  username     String   @unique
  avatarUrl    String?  @map("avatar_url")
  tier         String   @default("free") // 'free' | 'paid'
  createdAt    DateTime @default(now()) @map("created_at")

  follows    TopicFollow[]
  photos     Photo[]
  photoLikes PhotoLike[]

  @@map("accounts")
  @@schema("user")
}

// ─── topic schema ─────────────────────────────────────────────

model Topic {
  id            String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  name          String
  slug          String   @unique
  coverImageUrl String?  @map("cover_image_url")
  createdAt     DateTime @default(now()) @map("created_at")

  follows TopicFollow[]
  photos  Photo[]

  @@map("topics")
  @@schema("topic")
}

model TopicFollow {
  userId     String   @map("user_id") @db.Uuid
  topicId    String   @map("topic_id") @db.Uuid
  followedAt DateTime @default(now()) @map("followed_at")

  user  Account @relation(fields: [userId], references: [id], onDelete: Cascade)
  topic Topic   @relation(fields: [topicId], references: [id], onDelete: Cascade)

  @@id([userId, topicId])
  @@map("follows")
  @@schema("topic")
}

// ─── media schema ─────────────────────────────────────────────

model Photo {
  id         String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  topicId    String   @map("topic_id") @db.Uuid
  uploadedBy String   @map("uploaded_by") @db.Uuid
  imageUrl   String   @map("image_url")
  caption    String?
  createdAt  DateTime @default(now()) @map("created_at")

  topic    Topic      @relation(fields: [topicId], references: [id], onDelete: Cascade)
  uploader Account    @relation(fields: [uploadedBy], references: [id], onDelete: Cascade)
  likes    PhotoLike[]

  @@map("photos")
  @@schema("media")
}

model PhotoLike {
  userId  String   @map("user_id") @db.Uuid
  photoId String   @map("photo_id") @db.Uuid
  likedAt DateTime @default(now()) @map("liked_at")

  user  Account @relation(fields: [userId], references: [id], onDelete: Cascade)
  photo Photo   @relation(fields: [photoId], references: [id], onDelete: Cascade)

  @@id([userId, photoId])
  @@map("photo_likes")
  @@schema("media")
}
```

### Task 7: Seed topics

**Files:**

- [ ] **Step 1: Create seed file**

Create `api/prisma/seed.ts`:
```typescript
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient({
  log: ['query', 'info', 'warn', 'error'],
});

async function main() {
  const topics = [
    { name: 'Nature', slug: 'nature', coverImageUrl: null },
    { name: 'Architecture', slug: 'architecture', coverImageUrl: null },
    { name: 'Travel', slug: 'travel', coverImageUrl: null },
    { name: 'Food', slug: 'food', coverImageUrl: null },
    { name: 'Street Photography', slug: 'street-photography', coverImageUrl: null },
  ];

  for (const topic of topics) {
    await prisma.topic.upsert({
      where: { slug: topic.slug },
      update: {},
      create: topic,
    });
  }

  console.log('Seeded topics:', topics.map((t) => t.slug).join(', '));
}

main()
  .catch((e) => {
    console.error(e);
    process.exit(1);
  })
  .finally(() => prisma.$disconnect());
```
